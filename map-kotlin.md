# Задача: Разработка мобильного приложения на Kotlin

## Дата: 18-12-2025

## Цель

Создать нативное мобильное приложение на Kotlin для Android, которое будет предоставлять функционал веб-версии карты с учетом особенностей мобильной платформы.

## Анализ текущего веб-приложения

### Основной функционал

1. **Карта и маркеры:**

   - Отображение зданий, подразделений, точек доставки на карте
   - Кластеризация маркеров
   - Интерактивные маркеры с Popup
   - Построение маршрутов между точками
   - Обход препятствий (зданий) при построении маршрутов

2. **Управление данными:**

   - CRUD операции для зданий, этажей, подразделений
   - CRUD операции для точек доставки (warehouses)
   - Управление сотрудниками
   - Управление предметами, киосками, машинами, постаматами

3. **Маршрутизация:**

   - Построение пешеходных маршрутов с учетом зданий
   - Расчет расстояния маршрута
   - Выбор точек отправления и назначения
   - Отображение информации об ответственных сотрудниках

4. **Авторизация и права доступа:**
   - Роли пользователей
   - Права на редактирование карты
   - Разрешения на различные операции

### API Endpoints (из анализа reducers.ts)

**Здания (Buildings):**

- `GET /map/buildings` - получить все здания

**Этажи (Floors):**

- `POST /map/building/floor` - добавить этаж к зданию
- `DELETE /map/building/floor` - удалить этаж

**Подразделения (Departments):**

- `GET /map/departments` - получить все подразделения
- `POST /map/building/floor/department` - добавить подразделение к этажу
- `DELETE /map/building/floor/department` - удалить подразделение

**Точки доставки (Warehouses/Deliveries):**

- `GET /map/deliveries/post` - получить все точки доставки
- `POST /map/deliveries/post` - добавить точку доставки
- `PUT /map/deliveries/post` - обновить точку доставки
- `DELETE /map/deliveries/post` - удалить точку доставки

**Сотрудники (Employees):**

- `GET /map/employee` - получить всех сотрудников

**Предметы (Items):**

- `GET /map/building/item` - получить все предметы

**Другие сущности:**

- Киоски (Kiosks)
- Машины (Machines)
- Постаматы (Postamats)
- Места (Places)

### Типы данных

Основные модели данных:

- `Building` - здание с координатами (полигон), этажами
- `BuildingFloor` - этаж здания
- `FloorDepartment` - подразделение на этаже
- `Warehouse` - точка доставки с координатами, этажом, ответственными
- `Employee` - сотрудник (FIO, position, mdmcode)
- `RouteSelection` - выбор точки для маршрута

## Архитектура мобильного приложения

### Архитектурный паттерн

**Рекомендуется: MVVM (Model-View-ViewModel) + Clean Architecture**

```
app/
├── data/
│   ├── api/          # API клиенты (OkHttp)
│   ├── database/     # Локальная БД (Room)
│   ├── repository/   # Репозитории (Реализация)
│   └── models/       # Data модели (DTO)
├── domain/
│   ├── entities/     # Бизнес-сущности
│   ├── usecases/     # Use cases
│   └── repository/   # Интерфейсы репозиториев
├── presentation/
│   ├── ui/
│   │   ├── map/      # Экран карты (XML layouts)
│   │   ├── markers/  # Компоненты маркеров
│   │   └── routes/   # Построение маршрутов
│   ├── viewmodels/   # ViewModels
│   └── adapters/     # RecyclerView адаптеры
└── di/               # Dependency Injection (Koin/Hilt)
```
